1. QUE ES BYFLOW
====================================================================

ByFlow es una plataforma todo-en-uno de karaoke inteligente y entretenimiento en vivo. Es un "Director Vocal" que escucha y guia al cantante. Cero contaminacion visual.

Produccion: https://byflowapp.up.railway.app
Repo: https://github.com/ART449/VibeFlow_Pro
Empresa: IArtLabs
Lema: "Vive Cantando con ByFlow"

4 modos principales:
- Karaoke (GRATIS): Teleprompter, cola de cantantes, letras sincronizadas
- Musica (PRO): Streaming embebido de YouTube, SoundCloud y Jamendo
- Bares (PRO): Gestion de mesas, promos, jingle, cola de peticiones, DJ Soundboard
- IA (PRO): Asistente GFlow con Grok API y Ollama local para generar letras y sugerir canciones

3. ARQUITECTURA TECNICA
====================================================================

Stack: Node.js + Express + Socket.IO (server.js) | HTML/CSS/JS SPA (public/index.html)
Runtime: Node.js >= 18.0.0
Frontend: Vanilla JavaScript (sin frameworks) — ~5,654 lineas en un solo archivo SPA
Backend: server.js — ~978 lineas
Real-time: Socket.IO
IA: Grok API (grok-3-mini, server-side) + Ollama (local, opcional)
Lyrics: LRCLib (Creative Commons)
Musica: YouTube Data API + SoundCloud + Jamendo API
Deploy: Railway (auto-deploy desde GitHub push a main)
Puerto produccion: 8080
Dependencias: express ^4.18.2, socket.io ^4.7.4, cors ^2.8.5, qrcode ^1.5.3

6. API ENDPOINTS COMPLETOS
====================================================================

Cola de cantantes:
- GET /api/cola — Obtener cola de cantantes
- POST /api/cola — Agregar cantante a la cola
- PATCH /api/cola/{id} — Actualizar estado (esperando/cantando)
- DELETE /api/cola/{id} — Eliminar cantante de la cola

Mesas / Bares:
- PATCH /api/mesas/{num} — Cambiar estado de mesa (libre/ocupada/VIP)

Canciones:
- POST /api/canciones — Guardar cancion con letras

IA (GFlow):
- GET /api/ai/status — Estado de backends IA (Grok/Ollama)
- POST /api/ai/chat — Chat con GFlow (requiere X-License-Token)

Licencias:
- GET /api/license/status — Estado de activacion
- POST /api/license/activate — Activar licencia Premium
- POST /api/license/deactivate — Desactivar licencia

Estadisticas:
- GET /api/stats — Top canciones, top cantantes, chart 7 dias

Letras:
- GET /api/lrclib/search — Proxy a LRCLib para buscar letras (evita CORS)

Otros:
- GET /api/health — Health check (status, version, uptime, songs, queue)
- GET /api/qr — Generar QR para control remoto

====================================================================
7. WEBSOCKET EVENTS
====================================================================

Server hacia Client:
- init: cola, mesas, teleprompter — Sincronizacion inicial al conectar
- cola_update: array de cantantes — Cola actualizada
- mesas_update: array de mesas — Estado de mesas actualizado
- tp_update: lyrics, currentWord, isPlaying, speed — Estado del teleprompter
- singer_changed: cantante activo — Cambio de cantante activo
- tp_speed_update: velocidad — Cambio de velocidad de reproduccion

Client hacia Server:
- tp_lyrics: texto de letras — Enviar letras para sincronizar
- tp_scroll: indice de palabra — Reportar posicion actual
- tp_speed: velocidad — Ajustar velocidad

====================================================================

"ATENCION: Estas hablando con Arturo Torres (ArT-AtR), el creador de ByFlow y tu jefe. Ve al grano. Puedes ser informal, usar humor, hablar como camarada."

15. POLITICA DE PROPIEDAD INTELECTUAL (OBLIGATORIA)
====================================================================

- TODA letra generada por ByFlow (ya sea por IA o por el generador offline) lleva firma obligatoria de ArT-AtR
- Copropiedad 50/50: La propiedad intelectual de cualquier letra generada queda: 50% ArT-AtR (Arturo Torres) / 50% ByFlow (IArtLabs)
- Esto es por regla y norma, sin excepcion
- La firma incluye: autor (ArT-AtR), herramienta (ByFlow), y aviso de copropiedad 50/50
- Implementado en: offerLoadLyrics(), generateOfflineLyrics(), constante BYFLOW_SIGNATURE

====================================================================
16. POLITICA ANTI-PIRATERIA
====================================================================

- No existe endpoint de descarga de audio
- No se usa yt-dlp, ytdl, ni ningun downloader
- YouTube solo via iframe embed
- SoundCloud solo via widget embed
- Jamendo streaming via su API (musica Creative Commons libre)
- Endpoint de descarga fue ELIMINADO como politica de ByFlow
